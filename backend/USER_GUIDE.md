# AI Job Copilot 使用文档

## 1. 产品用途

AI Job Copilot 是一个面向个人求职流程的 AI 面试辅助工具。

它可以帮助你完成：

- 投递前分析 JD，判断岗位匹配度和面试风险。
- 面试前基于历史复盘生成准备方案。
- 面试中或面试前快速生成回答思路。
- 面试后把复盘沉淀成 Job Memory。
- 下次面试时通过 RAG 召回历史经验，复用自己的真实经历。

核心原则：

> 不让 AI 替你编经历，而是让 AI 帮你整理、检索和复用自己的真实经历。

## 2. 使用入口

当前项目有两个入口：

- 微信小程序：用于手机端体验核心流程。
- H5 页面：用于浏览器访问和调试。

当前内测阶段使用邀请码访问。第一次打开小程序时，需要输入邀请码。验证成功后，后续请求会自动携带邀请码，不需要每次重新输入。

如果手机端提示连接失败，但手机浏览器可以打开后端健康检查 JSON，说明服务器本身可用，问题通常来自小程序环境限制，例如 HTTP/IP 地址、合法域名校验或真机调试模式。

## 3. 首次使用

### 3.1 输入邀请码

打开小程序后，如果还没有登录，会进入邀请码页。

操作步骤：

1. 输入内测邀请码。
2. 点击确认。
3. 验证成功后进入首页。

邀请码用于：

- 防止陌生人消耗 AI API。
- 区分不同试用者的数据空间。
- 避免别人的 Job Memory 污染你的历史记忆。

邀请码不是完整账号系统，也不建议公开发布。

### 3.2 检查系统状态

进入“我的”页面，可以查看系统状态。

正常状态应包含：

- 后端服务正常。
- AI Key 已配置。
- Job Memory 已启用。
- Embedding 已启用。
- 用户隔离已启用。

如果系统状态异常，可以先检查后端健康接口：

```text
GET /api/health
```

## 4. 分析 JD

### 4.1 适用场景

当你看到一个岗位，但不确定是否值得投、怎么准备、风险点在哪里时，可以使用 JD 分析。

### 4.2 使用步骤

1. 进入 JD 输入页。
2. 粘贴岗位 JD 文本，或上传岗位截图进行 OCR。
3. 确认自己的简历信息已经填写或保存在系统中。
4. 点击生成分析。

### 4.3 输出内容

系统会生成：

- 岗位摘要
- 匹配分
- 主要匹配点
- 主要风险点
- 高风险面试雷达
- 缺失能力
- 简历优化建议
- 打招呼语

### 4.4 高风险面试雷达

在 JD 分析结果页，“高风险面试雷达”里的问题可以点击。

点击后会跳转到“问一问 AI”，并自动填入这道面试题。你可以直接生成回答思路。

推荐链路：

```text
JD 分析
-> 看到高风险问题
-> 点击问题
-> 问一问 AI 生成回答思路
-> 面试后复盘沉淀
```

## 5. 问一问 AI

### 5.1 适用场景

当你想准备某个具体面试问题时，可以使用“问一问 AI”。

例如：

- “你为什么适合这个岗位？”
- “你没有硬件 AI 经验怎么办？”
- “这个项目怎么证明不是玩具？”
- “你在项目中最大的卡点是什么？”

### 5.2 使用步骤

1. 进入“问一问 AI”。
2. 输入面试问题。
3. 选择轮次、回答风格和回答时长。
4. 点击生成回答思路。

如果你是从 JD 分析结果页或回答结果页点击追问进入，问题会自动填入，并保留上一轮的轮次、风格和时长。

### 5.3 输出内容

系统会生成：

- 30 秒短答
- 完整答法
- 可引用证据
- 不要这样说
- 可能追问
- 本次参考记忆

如果系统召回了历史 Job Memory，会在结果中展示本次参考的历史记忆。这样可以知道 AI 的回答参考了哪些真实经历。

### 5.4 可能追问

回答结果页里的“可能追问”可以点击。

点击后会回到“问一问 AI”，并自动填入这个追问。你可以继续生成回答思路。

这让问答不再是一次性结果，而是可以连续追问的面试准备 Copilot。

## 6. 面试后复盘

### 6.1 适用场景

每次面试结束后，都建议把复盘保存下来。

可以粘贴：

- 自己写的面试复盘。
- 面试问题清单。
- 录音转文字结果。
- ASR 原始文本。
- 聊天记录里整理出的面试反馈。

### 6.2 使用步骤

1. 进入复盘页。
2. 填写公司、岗位、轮次和结果。
3. 粘贴复盘内容。
4. 点击生成 Job Memory。
5. 确认生成结果后保存。

### 6.3 Job Memory 会保存什么

系统会把复盘结构化为：

- 公司
- 岗位
- 轮次
- 结果
- 复盘摘要
- 面试官问题
- 考察意图
- 回答卡点
- 下次策略
- 可复用证据
- 标签

保存后，这条 Job Memory 会进入历史记忆库，并用于后续 RAG 召回。

## 7. 历史记忆

### 7.1 查看历史

进入历史页，可以看到已经保存的 Job Memory。

每张卡片展示：

- 公司
- 岗位
- 轮次
- 面试结果
- 摘要
- 标签

### 7.2 展开详情

点击历史卡片可以展开详情，查看：

- 面试问题
- 回答卡点
- 下次策略
- 可复用证据

这部分是后续 RAG 检索的核心材料。

### 7.3 删除重复记录

如果重复上传了复盘，或某条记录质量不好，可以在历史页删除对应 Job Memory。

删除后：

- 这条记录不会再出现在历史列表。
- 后续问答和面试准备不会再召回它。
- 只能删除当前邀请码对应用户空间下的数据。

## 8. 基于历史记忆准备面试

在 JD 分析结果页，可以使用“历史记忆准备”能力。

系统会结合当前 JD 和历史 Job Memory，生成：

- 本轮准备重点
- 历史风险
- 可能问题
- 回答策略
- STAR 回答提醒
- 练习计划
- 一分钟 Pitch

这里的 STAR 是一种面试回答结构：

- Situation：背景是什么。
- Task：当时你的任务是什么。
- Action：你具体做了什么。
- Result：最后结果如何。

建议把自己的真实项目经历整理成 STAR 结构，后续问答会更稳定。

## 9. 数据隔离与隐私

当前内测版本通过邀请码做轻量用户隔离。

每个邀请码可以对应一个独立 `user_id`。后端会按 `user_id` 隔离：

- 复盘保存
- 历史列表
- 删除权限
- RAG 召回

注意事项：

- 不要把真实 `.env` 文件上传到 GitHub。
- 不要把 DeepSeek、Supabase、Embedding 的真实 Key 发给别人。
- 不建议在公开场合暴露邀请码。
- 云端服务会接收 JD、简历文本和复盘内容，用于 AI 分析和记忆检索。
- 如果给别人试用，最好为对方单独配置邀请码。

## 10. 常见问题

### 10.1 输入邀请码后提示连接后端失败

可能原因：

- 后端服务没有启动。
- 服务器防火墙没有放行端口。
- 小程序环境不允许访问 HTTP/IP 地址。
- 微信开发者工具没有勾选“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
- 手机端没有使用真机调试或体验版合法域名未配置。

可以先用手机浏览器打开健康检查地址。如果能看到 JSON，说明服务器外网可访问。

### 10.2 手机浏览器能看到 JSON，但小程序仍然失败

这通常不是服务器问题，而是微信小程序限制。

正式可用的小程序通常需要：

- HTTPS 域名。
- 微信公众平台配置 request 合法域名。
- 小程序体验版或正式版。

开发阶段可以使用微信开发者工具的“不校验合法域名”选项，或使用真机调试。

### 10.3 电脑关了还能用吗

如果后端已经部署在腾讯云服务器上，电脑关了也可以用。

前提是：

- 云服务器在运行。
- PM2 中的后端进程在线。
- API 地址指向腾讯云服务器。
- 小程序当前环境允许访问该地址。

### 10.4 复盘粘贴一大段无法识别怎么办

建议：

- 尽量保留面试官问题和你的回答。
- 如果是 ASR 文本，先删除明显无关的口水词。
- 过长内容可以分段保存。
- 如果重复保存了，可以到历史页删除质量较差的记录。

### 10.5 为什么 AI 没有引用我的历史经历

可能原因：

- 还没有保存过 Job Memory。
- 当前问题和历史复盘相似度不高。
- 历史复盘内容太短，缺少可检索信息。
- Embedding 或 Supabase 配置异常。

可以先进入“我的”页面检查 Job Memory 和 Embedding 状态。

## 11. 推荐使用流程

日常求职时，推荐按这个顺序使用：

```text
1. 保存或更新简历信息
2. 粘贴 JD 做岗位分析
3. 查看高风险面试雷达
4. 点击问题进入问一问 AI
5. 面试后粘贴复盘
6. 保存为 Job Memory
7. 下次 JD 分析时用历史记忆准备
```

这个闭环跑得越多，Job Memory 越丰富，后续回答越容易贴近你的真实经历。
